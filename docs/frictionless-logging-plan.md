# Frictionless Logging (상태 표시 우선 UX) 개편 플랜

## 1. 배경 및 진단
현재 Ottline의 기록 UX(`QuickLogCard`)는 "제목을 검색하고 폼을 꽉 채워야 하는" 형태의 높은 인지 부하를 주고 있습니다. 특히 첫 진입 사용자에게 이 과정은 무겁게 느껴져 **"첫 기록 행동(1st Action)"** 자체의 발생률이 낮을 가능성이 높습니다.
이에, 기록이 아니라 가장 가벼운 단위인 **"상태 표시(Status)"를 1초 만에 완료**하도록 UX의 핵심 축을 변경합니다.

또한, Ottline은 웹뿐만 아니라 PWA, TWA(Android), 브라우저 확장 프로그램 등 **멀티 플랫폼(Multi-platform)**으로 제공되므로, 저장 위치 안내 시 "브라우저"라는 종속적인 단어 대신 **"이 기기(This device)"**라는 플랫폼 중립적인 용어를 사용합니다.

## 2. 목표
- **첫 상태 표시율(First Status Action Rate)** 극대화.
- 기록 완료(저장)의 심리적 장벽 제거 (단 한 번의 터치로 저장).
- "로그인 없는 로컬 퍼스트 저장"에 대한 신뢰도 있는 안내 (현재 기기 저장 명시).
- 개인 저장과 공개 타임라인 공유의 완벽한 맥락 분리.

## 3. 핵심 변경 사항

### Phase 1: 상태 우선(State-First) UI 개편 (`QuickLogCard.tsx`)
**기존:** 
작품 검색 -> [상태, 별점, 플랫폼, 장소, 상황, 날짜, 노트] 전체 폼 노출 -> 저장 및 공유

**개편안 (Progressive Disclosure):**
1. **검색 단계:** [영상/도서] 탭 + 검색창만 노출.
2. **상태 선택 단계 (핵심):** 검색 결과 선택 시, 썸네일/타이틀과 함께 **거대한 상태 버튼 3개**만 최상단에 노출.
   - 영상: `[봤어요]`, `[보는 중]`, `[보고 싶어요]`
   - 도서: `[읽었어요]`, `[읽는 중]`, `[읽고 싶어요]`
   - 하단 안내 문구: `"로그인 없이 이 기기에 먼저 저장돼요."`
3. **원클릭 저장:** 상태 버튼 중 하나를 누르는 순간 즉시 **로컬 DB 저장(장소/상황 등 기본값은 비움) 및 Outbox 스케줄링 완료**.

### Phase 2: 개인 기록과 공개 공유의 분리 및 후속 입력
- 상태 버튼을 눌러 **이미 저장이 완료된 상태**에서, 그 아래로 "더 남길 내용이 있나요?" (별점, OTT, 메모 등) 형태의 후속 입력 폼을 부드럽게 펼쳐 보여줍니다.
- 공유 옵션(`shareToDiscussion`, `shareCard`)은 "저장" 버튼과 완전히 분리된 맥락인 **"선택 사항: 라운지에 감상 공유하기"** 영역으로 하단에 재배치하여, "저장 = 공개"라는 오해를 원천 차단합니다.
- 내 타임라인이나 홈의 저장된 기록(LogCard)에서 메모(노트)가 없는 경우 `+ 별점·메모 남기기` 뱃지를 제공해 추가 입력을 유도하고, 메모가 있는 경우 `[ 함께 하는 기록 남기기 ]` 버튼을 띄워 곧바로 라운지에 공유할 수 있도록 합니다.

### Phase 3: 다국어(i18n) 번역 파일 업데이트 및 문구 교정
- `ko.json`, `en.json`에 새로운 안내 문구를 추가합니다.
  - 예: `"로그인 없이 이 기기에 먼저 저장돼요."` / `"Saved locally to this device first without login."`
- "한 줄만 적어두면 타임라인이 쌓여요" ➔ "버튼 한 번 클릭으로 타임라인이 쌓여요"로 변경하여 바뀐 기획 방향과 싱크를 맞춥니다.
- "공유 이미지 카드"라는 용어를 "공유 포토 카드"로 전면 교체합니다.

### Phase 4: 크로스 브라우징 및 UI 디테일 대통일
- 웹 앱 전체(홈 퀵로그, 타임라인 필터 바, 함께/퍼블릭 메뉴의 검색 바)에서 쓰이는 **모든 입력창 및 셀렉트 박스의 높이를 40px(h-10)로 완벽하게 통일**하여 시각적 안정감을 확보했습니다.
- 모바일에서 과도하게 줄바꿈되던 폼 요소(필터 3종 등)를 가로 1줄 및 스크롤(`flex-nowrap overflow-x-auto no-scrollbar`)로 고정했습니다.
- 크롬 PC 등에서 요소들이 지나치게 붙어 나오거나 `flex` 속성에 의해 버튼 높이가 찌그러지던 현상들을 강제 마진과 여백, 명시적인 높이값으로 원천 차단했습니다.

## 4. 백엔드 및 기타 보완 사항
- **CORS 설정 완화:** ngrok 등 다양한 임시 URL 또는 모바일 환경에서의 기기 등록(`register`) 요청 실패 시 런타임 에러를 방지하고 안정적인 동기화가 유지되도록 백엔드의 `WebConfig.java` CORS 허용 및 프론트엔드 에러 로깅을 강화했습니다.
- **TMDB 상세 정보 매핑:** `QuickLogCard`에서 작품 선택 시 검색 결과에는 없는 '장르, 감독, 주연' 데이터를 즉시 불러와 포스터 우측에 세련되게 노출하도록 백엔드에 엔드포인트를 신설하고 통신을 연결했습니다.

## 5. 지표(Metrics) 관점의 기대 효과
이 개편을 통해 "방문자 중 몇 %가 첫 번째 콘텐츠 상태를 남기는가?"(`첫 상태 표시율`) 지표가 비약적으로 상승할 것으로 기대하며, 이는 향후 "과거 기록 복원" 등의 재방문 넛지(Nudge)를 실행하기 위한 강력한 데이터 기반이 됩니다.
