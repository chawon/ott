# 서비스 UX 리뷰 (2026-02-06)

## 요약
- 전체적으로 `기록 저장`의 핵심 플로우는 빠르고 명확하다.
- 다만 실제 사용자 경험에서 이탈을 만드는 지점은 세 가지다.
- 1) 용어/언어 톤 혼합, 2) 고급 옵션 노출 과다, 3) 실패/동기화 상태 가시성 부족.

## 적용 현황 (2026-02-06 반영)
- 완료: 사용자 노출 카피 한국어 통일
- 완료: 검색 패널 문구 정합성 개선(책 모드에서 “비디오” 표현 제거)
- 완료: 공유 옵션 라벨 개선
- 완료: 저장 버튼 근처에 공유 체크박스 배치
- 완료: 공유 옵션 문구 변경
- `함께 하는 기록 남기기`
- `공유 이미지 카드 만들기`
- 완료: `저장 후 함께 하기` 라벨에 아이콘 추가
- 보류: QuickLog `자세히 기록` 접기 실험
- 이유: 체감 개선이 크지 않아 원복
- 보류: 전역 동기화 상태 표시
- 이유: 화면 노출 피로도가 높아 원복

## 장점
- 홈에서 바로 기록 가능한 구조가 명확함 (`apps/web/app/page.tsx:164`)
- 저장 직후 피드백(배너 + 타임라인 이동)이 빠름 (`apps/web/components/QuickLogCard.tsx:1160`)
- 책/영상 모드 전환이 일관된 컴포넌트로 구현됨 (`apps/web/components/QuickLogCard.tsx:851`)
- 설정에서 CSV 내보내기까지 한 번에 접근 가능 (`apps/web/app/account/page.tsx:280`)

## 주요 UX 이슈 (우선순위 순)

### 1) P1: 언어/톤 일관성 부족으로 신뢰감 저하
- 근거:
  - 로딩/에러/빈 상태에 영어 혼재: `Loading…`, `Error`, `Not found` (`apps/web/app/page.tsx:213`, `apps/web/app/title/[id]/page.tsx:541`)
  - 검색 패널에 `LOADING...`, `EMPTY` 노출 (`apps/web/components/TitleSearchBox.tsx:193`, `apps/web/components/TitleSearchBox.tsx:199`)
  - 레트로 책 모드에서도 “비디오들” 문구 노출 (`apps/web/components/TitleSearchBox.tsx:190`)
- 영향:
  - 서비스 완성도 체감 하락, 특히 첫 방문자 신뢰 저하
- 제안:
  - 사용자 노출 카피를 한국어 톤으로 통일
  - `contentType` 기준 동적 카피 분기(책/영상 명시)

### 2) P1: QuickLog에 선택지가 많아 “10초 저장” 가치가 약해짐
- 근거:
  - 기본 폼에서 상태/평점/장소/누구와/플랫폼/날짜/메모/공개공유/공유카드까지 동시 노출 (`apps/web/components/QuickLogCard.tsx:924`)
- 영향:
  - 초보 사용자는 저장 전 인지 부하가 큼
  - 핵심 액션(저장) 이전에 선택 피로 발생
- 제안:
  - 기본: `상태 + 메모`만 먼저 노출
  - 나머지는 “자세히 기록” 접이식 섹션으로 분리
  - 저장 후 상세 편집 유도
- 현재 판단:
- 접이식 실험은 원복했고, 대신 저장 영역 근처 공유 옵션의 이해도 개선에 집중

### 3) P1: 동기화 상태가 사용자에게 보이지 않아 불안 유발
- 근거:
  - 오프라인/동기화 실패 시 사용자에게 명시 배지/토스트가 거의 없음
  - 로컬 우선 저장은 되지만 “서버 반영 여부”가 화면에서 구분되지 않음
- 영향:
  - “저장됐는지/다른 기기에 반영됐는지” 불명확
- 제안:
  - 헤더 또는 카드에 `동기화 중/완료/실패` 상태 표시
  - 실패 시 재시도 CTA 제공
- 현재 판단:
- 전역 노출 방식은 원복, 필요 시 실패 상황에서만 제한적으로 노출하는 방향 재검토

### 4) P2: 정보구조(IA) 관점에서 설정 페이지 밀도 과다
- 근거:
  - 페어링, 계정 식별자, 로컬 초기화, 내보내기, 기기관리, 계정 초기화가 한 화면에 집중 (`apps/web/app/account/page.tsx:213`)
- 영향:
  - 파괴적 액션(초기화/해제)의 맥락이 약함
  - 실수 가능성 상승
- 제안:
  - 섹션 그룹화: `계정 연결`, `데이터 관리`, `보안/기기`
  - 파괴적 액션은 별도 경고 스타일 및 2차 확인 강화

### 5) P2: 타임라인 필터 UX에서 의미 충돌
- 근거:
  - `콘텐츠=전체`일 때 상태 필터 disabled (`apps/web/components/FiltersBar.tsx:131`)
  - 사용자 관점에서는 “전체에서도 상태 필터링”을 기대할 가능성 높음
- 영향:
  - 필터 동작 모델 학습 비용 증가
- 제안:
  - 전체에서도 상태 필터 허용
  - 또는 비활성 이유를 툴팁/도움말로 명확히 설명

### 6) P3: 공유 옵션의 의미 구분이 직관적이지 않음
- 근거:
  - “함께 기록 공유”와 “공유 카드 만들기”가 동일 위계로 병치 (`apps/web/components/QuickLogCard.tsx:1128`)
- 영향:
  - 공개 게시/이미지 생성의 차이를 초보 사용자가 즉시 이해하기 어려움
- 제안:
  - 라벨 보강:
  - `함께 기록 공유(공개)` / `공유 카드 만들기(이미지)`
  - 첫 사용 시 1회 도움말 제공

## 우선 실행안 (2주)

1. 카피 통일 스프린트
- 사용자 노출 영어 문구 제거
- 책/영상 모드별 문구 정합성 수정

2. QuickLog 단순화 실험
- 기본 폼 필드 축소 A/B
- KPI: 저장 완료율, 첫 저장까지 시간

3. 동기화 상태 가시화
- 글로벌 sync indicator + 실패 재시도 버튼
- KPI: “저장 안 됨” 관련 문의/이탈 감소

4. 설정 IA 재배치
- 파괴적 액션 분리 + 경고 강화
- KPI: 초기화/기기해제 오조작 감소

- `sync failure retry success rate`
