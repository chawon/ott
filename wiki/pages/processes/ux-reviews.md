## 총평

핵심 플로우(기록 저장 → 타임라인 확인)는 명확하고 로컬 우선 저장 구조가 사용성에 유리하다. 주요 개선 필요 영역: 접근성, 카피 일관성, 모션 접근성, 설정 IA.

## 주요 이슈

### P1: 접근성 — 폼 레이블 누락 (2026-02-10)
- `div` 텍스트 + `select/input` 조합이나 연결된 `<label>`이 없는 필드 다수
- 영향: 스크린리더 사용 시 필드 목적 인지 어려움
- **해결:** 각 입력 요소에 `id` + `<label htmlFor>` 연결, 레이아웃 어려운 경우 `aria-label`
- 관련 파일: `QuickLogCard.tsx:942`, `FiltersBar.tsx:136`, `account/page.tsx:245`

### P1: 접근성 — 아이콘 전용 버튼 접근 이름 누락 (2026-02-10)
- 선택 항목 해제 버튼(X 아이콘)에 접근 이름 없음
- **해결:** `aria-label="선택한 작품 해제"` 추가

### P1: 접근성 — 전역 Skip Link 부재 (2026-02-10)
- 루트 레이아웃에 `skip to main content` 링크 없음
- **해결:** `<body>` 초반 스킵 링크 추가

### P1: 언어/톤 일관성 부족 (2026-02-06)
- 로딩/에러/빈 상태에 영어 혼재: `Loading…`, `Error`, `Not found`
- 검색 패널에 `LOADING...`, `EMPTY`
- **해결:** 사용자 노출 카피 한국어 톤으로 통일 (→ [[ux-copy]])

### P2: 모션 접근성 — `prefers-reduced-motion` 대응 부족 (2026-02-10)
- `animate-bounce`, `animate-in` 등 강한 애니메이션 사용
- **해결:** `motion-reduce:animate-none` 또는 조건부 렌더

### P2: 이모지 아이콘 사용 (2026-02-10)
- 코멘트 수 표시에 `💬` 이모지 사용
- **해결:** `MessageCircle` 같은 SVG 아이콘으로 교체 (완료)

### P2: 설정 페이지 밀도 과다 (2026-02-06)
- 페어링, 계정 식별자, 로컬 초기화, 내보내기, 기기관리가 한 화면에 집중
- **제안:** 섹션 그룹화 (`계정 연결`, `데이터 관리`, `보안/기기`)

### P2: 타임라인 필터 UX 의미 충돌 (2026-02-06 & 2026-02-10)
- `콘텐츠=전체`일 때 상태 필터 disabled → 사용자 기대와 불일치
- **제안:** 상태 필터 항상 활성화하거나 비활성 사유 명시

## 완료된 개선 사항

### 2026-02-06 반영
- 사용자 노출 카피 한국어 통일
- 검색 패널 문구 정합성 개선 (책 모드에서 "비디오" 표현 제거)
- 공유 옵션 라벨 개선 + 저장 버튼 근처 배치

### 2026-02-10 반영
- 온보딩 CTA: `보기` → `타임라인 보기`
- 공개 상세 페이지 카피 국문화 (`INVALID ROUTE`, `LOADING...`, `Error`, `NOT FOUND` 제거)
- 타입 라벨 `Movie/Series/Book` → `영화/시리즈/책`
- 이모지 아이콘 제거 (`💬` → 댓글 텍스트)
- 댓글 패널 오류/로딩 문구 정리

### 보류 결정
- QuickLog `자세히 기록` 접기 실험 → 원복 (체감 개선 미미)
- 전역 동기화 상태 표시 → 원복 (화면 노출 피로도)
