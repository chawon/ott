# 포인트 색상 역할 정리

## 목표와 범위

오렌지 `#FF9933`은 기록 완료와 저장 같은 최종 주요 행동, 포커스 신호, 현재 탐색 위치를 알리는 작은 지속 표시로 사용한다. 탐색, 선택, 상태, 지표, 보조 링크에 반복되던 오렌지 채움과 텍스트는 기존 따뜻한 중립색이나 절제된 워드마크 네이비로 낮추되, 현재 위치에는 얇은 선이나 짧은 표시를 남겨 브랜드 존재감이 사라지지 않게 한다.

이번 변경은 웹과 iOS 네이티브 앱의 주요 탐색·기록·리포트 화면과 계정·서비스 소개의 일반 정보 및 선택 표면을 포함한다. 카피, 레이아웃, 모션, API, DTO, 데이터베이스 스키마, 동기화 계약에는 영향이 없다. 공유·복구 카드로 내보내는 브랜드 산출물의 재디자인은 이번 범위에 포함하지 않는다.

## 의미 역할

| 역할 | 색상과 사용 규칙 |
|---|---|
| `action` / `focus` | `#FF9933`. 최종 기록·저장 주요 버튼과 키보드/입력 포커스에만 사용한다. |
| `onAction` | `#0F0F0F`. 오렌지 채움 위의 텍스트와 아이콘에 사용한다. |
| `navigation` / `link` | 밝은 화면에서는 기존 워드마크 네이비 `#1E4D8C`을 텍스트·아이콘에 절제해 사용한다. 웹과 네이티브의 현재 탐색 위치에는 오렌지 채움이나 전체 오렌지 텍스트 대신 얇은 선 또는 짧은 표시를 지속적으로 노출한다. 어두운 화면에서는 기존 가독성 높은 중립 토큰과 같은 크기의 오렌지 표시를 사용한다. |
| `selection` | `#FEF9EE`, `#ECEBE9`, `#FAF5D7` 등 따뜻한 중립 배경과 near-black 텍스트를 사용한다. 선택 상태를 오렌지 채움으로 만들지 않는다. |
| `status` / `metric` / `metadata` | 기본은 near-black 또는 charcoal과 따뜻한 중립 배경이다. 일반 정보에 쓰인 purple, indigo, sky 등 팔레트 밖 강조색도 이 역할로 정리한다. |
| `brand exception` | `OTTLINE` 워드마크의 기존 파랑과 `ON THE TIMELINE` 서브타이틀의 오렌지는 그대로 유지한다. |

한 작업 영역에는 오렌지 주요 행동을 하나만 두는 것을 기본으로 한다. 포커스 링은 행동 색과 같은 오렌지를 사용할 수 있다. 지속적으로 보이는 오렌지 선택·탐색 채움과 반복 텍스트는 사용하지 않지만, 현재 탐색 위치를 나타내는 작은 오렌지 micro-accent는 예외로 둔다. 리캡 노티에서 리포트로 이동하는 CTA는 탐색 역할이므로 중립 또는 네이비로 처리하고, 리포트 안의 공유 버튼만 해당 화면의 단일 오렌지 행동으로 유지한다.

## 검증 시나리오

1. 웹 데스크톱의 `/ko`와 `/en` 홈에서 헤더 메뉴, 언어 선택, 첫 기록 패널, QuickLog 선택 상태가 중립 또는 네이비 역할을 사용하고 최종 기록·저장 행동만 오렌지 채움인지 확인한다. 현재 헤더 메뉴에는 얇은 오렌지 선이나 짧은 표시가 지속적으로 보여야 한다.
2. 웹 모바일의 `/ko`와 `/en` 홈·타임라인에서 하단 탐색 셸의 오렌지 테두리와 광택은 사라지되, 현재 탭의 작은 오렌지 표시가 보이는지 확인한다. 활성 아이콘·레이블과 긴 번역 문자열은 겹치거나 잘리지 않아야 한다.
3. 웹 데스크톱·모바일의 기록이 있는 타임라인과 리포트에서 반복 카드의 메타데이터와 KPI가 오렌지 패턴을 만들지 않고 포스터·커버보다 먼저 튀지 않는지 확인한다.
4. 웹에서 키보드로 탐색해 포커스 링이 사라지지 않았는지, 오렌지 채움 위 텍스트·아이콘이 `#0F0F0F`인지 확인한다.
5. 네이티브의 한국어·영어 기록하기, 탭 탐색, 리포트 화면을 라이트·다크 모드에서 확인한다. 선택 상태는 중립 역할을 사용하고 오렌지 채움은 최종 기록·저장에만 남기되, 현재 활성 탭에는 작은 오렌지 표시가 지속적으로 보여야 한다.
6. 웹과 네이티브 모두에서 파란 `OTTLINE` 워드마크와 오렌지 `ON THE TIMELINE` 서브타이틀이 유지되는지 확인한다.
