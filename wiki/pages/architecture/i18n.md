## 개요

**완료일:** 2026-03-06

Microsoft Store 등 글로벌 스토어 배포를 위해 한국어(`ko`)와 영어(`en`)를 지원하는 i18n 체계를 구축했다. 향후 추가 언어도 확장 가능한 구조다.

---

## 프론트엔드 구현 (Next.js App Router)

### 라이브러리 및 라우팅
- `next-intl` 사용
- URL 기반 언어 구분: `/ko/...`, `/en/...`
- Middleware: 브라우저 언어 감지 → 적절한 locale로 리다이렉트
- 번역 사전: `apps/web/messages/en.json`, `apps/web/messages/ko.json`

### 주요 변경
- `app/` 하위 모든 라우트를 `app/[locale]/`로 이동
- 공통 컴포넌트(Header, Footer, Navigation)와 핵심 화면(홈, 타임라인, 상세, QuickLog) 번역 완료
- `lib/utils.ts` 내 상태/장소/평점 라벨 생성 로직 다국어 지원, 기본값 영문으로 변경

---

## 백엔드 구현 (Spring Boot)

- 모든 핵심 Controller에서 `Accept-Language` 헤더 수신
- 프론트엔드 `lib/api.ts`에서 현재 locale을 전역 헤더로 전송
- TMDB API 호출 시 언어 파라미터 전달 → 해당 언어 검색 결과/상세 반환
- SyncService: 데이터 동기화 시 타이틀 정보를 해당 언어로 hydration

---

## 특이사항

- **날짜 포맷**: 언어 설정에 따라 `ko-KR` 또는 `en-US` 형식 동적 적용
- **UUID Fallback**: `crypto.randomUUID` 미지원 환경(HTTP 등)을 위해 `Math.random` 기반 폴백을 `lib/utils.ts`에 추가
- **언어 추가 방법**: `messages/<lang>.json` 파일 추가 + `i18n/routing.ts`의 `locales` 배열에 추가

## 관리자 분석 페이지 i18n 이슈

`/[locale]/admin/analytics` 접근이 preview 환경에서 실패하는 문제가 있었다.

**원인:** 프론트 `ADMIN_ANALYTICS_TOKEN` 환경변수가 preview에 주입되지 않으면 올바른 URL 토큰으로도 접근 불가

**해결 방향:**
- 프론트 env가 설정된 경우: 기존대로 토큰 불일치 시 `notFound()`
- 프론트 env가 비어 있으면: URL 토큰을 백엔드 `X-Admin-Token` 헤더로 전달
- 최종 권한 판단은 백엔드 `/api/admin/analytics/*`가 수행
