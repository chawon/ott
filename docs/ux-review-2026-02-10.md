# 서비스 UX/PWA 리뷰 (2026-02-10)

## 범위
- 프론트엔드: `apps/web`
- 핵심 화면: 홈(기록), 타임라인, 함께, 설정, 오프라인, PWA 설치 배너
- 제품 맥락: 일반 사용자가 영상/독서 기록을 남기고 타임라인으로 보는 Web Service + PWA

## 리뷰 기준
- `ui-ux-pro-max` 디자인 시스템 추천값을 기준선으로 사용
- 기준선 요약
- 패턴: 빠른 기록 중심(단순 입력, 명확한 CTA)
- 스타일: Flat, 고대비, 빠른 로딩, 군더더기 최소화
- UX 우선순위: 입력 레이블, 접근성 이름, 키보드 내비게이션, 대비, 오프라인 신뢰

## 총평
- 기록 저장과 타임라인 확인이라는 핵심 플로우는 명확하고, 로컬 우선 저장 + 동기화 구조가 사용성에 유리합니다.
- 다만 접근성(폼 레이블/아이콘 버튼 이름/스킵 링크), 카피 일관성, 모션 접근성에서 제품 완성도를 낮추는 지점이 확인됩니다.

## 강점
- 홈에서 즉시 기록 가능한 구조가 명확함: `apps/web/app/page.tsx`
- 저장 직후 피드백(배너 + 타임라인 이동)으로 완료감이 좋음: `apps/web/components/QuickLogCard.tsx:1188`
- 타임라인 필터 + CSV 내보내기까지 사용자 작업 흐름이 짧음: `apps/web/app/timeline/page.tsx`, `apps/web/app/account/page.tsx:292`
- 오프라인 페이지와 서비스워커가 기본 사용자 보호선 역할을 수행함: `apps/web/app/offline/page.tsx`, `apps/web/public/sw.js`

## 주요 이슈 (우선순위 순)

### 1) P1 접근성: 폼 레이블이 시각 텍스트에만 의존
- 근거
- `div` 텍스트 + `select/input` 조합으로 연결된 `<label>`이 없는 필드가 다수 존재
- `apps/web/components/QuickLogCard.tsx:942`
- `apps/web/components/FiltersBar.tsx:136`
- `apps/web/app/public/page.tsx:73`
- `apps/web/app/account/page.tsx:245`
- 영향
- 스크린리더 사용 시 필드 목적 인지가 떨어지고, 폼 탐색 효율이 크게 저하됨
- 개선
- 각 입력 요소에 `id`를 부여하고 `<label htmlFor>`로 연결
- 레이아웃상 라벨이 어려운 경우 `aria-label` 최소 보강

### 2) P1 접근성: 아이콘 전용 버튼의 접근 이름 누락
- 근거
- 선택 항목 해제 버튼이 아이콘(X)만 있고 접근 이름이 없음: `apps/web/components/QuickLogCard.tsx:930`
- 영향
- 보조기기 사용자에게 버튼 의미가 전달되지 않음
- 개선
- `aria-label="선택한 작품 해제"` 추가
- 키보드 포커스 스타일도 명시적으로 강화

### 3) P1 접근성: 전역 Skip Link 부재
- 근거
- 루트 레이아웃에 `skip to main content` 링크 없음: `apps/web/app/layout.tsx:42`
- 영향
- 키보드 사용자에게 헤더/네비게이션 반복 탐색 부담 발생
- 개선
- `<body>` 초반에 스킵 링크 추가 후 `main`에 연결

### 4) P2 카피/브랜드 일관성: 의미 없는 툴팁 및 언어 혼합
- 근거
- 레트로 토글 버튼 `title`이 `"???"`: `apps/web/components/AppHeader.tsx:70`
- 최신 UI에서 일부 영어 타입 라벨 사용(`Movie`, `Series`, `Book`): `apps/web/components/DiscussionList.tsx:105`
- 영향
- 제품 신뢰감 저하, 초반 학습 비용 증가
- 개선
- 토글 버튼 카피를 명확한 행동 문구로 교체
- 사용자 노출 타입 라벨 언어를 한국어로 통일

### 5) P2 디자인 품질 규칙 위반: 이모지 아이콘 사용
- 근거
- 코멘트 수 표시에 이모지(`💬`) 사용
- `apps/web/components/TitleSearchBox.tsx:251`
- `apps/web/components/DiscussionList.tsx:72`
- 영향
- 시각 톤 일관성 저하, 플랫폼별 렌더 편차 발생
- 개선
- `MessageCircle` 같은 SVG 아이콘으로 교체

### 6) P2 모션 접근성: `prefers-reduced-motion` 대응 부족
- 근거
- 강한 애니메이션(`animate-bounce`, `animate-in`) 사용: `apps/web/components/QuickLogCard.tsx:846`, `apps/web/components/QuickLogCard.tsx:1188`
- 코드베이스에서 `prefers-reduced-motion`/`motion-reduce` 적용 흔적 없음
- 영향
- 모션 민감 사용자에게 피로/불편 유발 가능
- 개선
- `motion-reduce:animate-none` 또는 조건부 렌더로 완화

### 7) P3 타임라인 필터 학습 비용
- 근거
- 콘텐츠가 `ALL`일 때 상태 필터가 비활성화됨: `apps/web/components/FiltersBar.tsx:131`
- 영향
- 사용자 기대(전체에서도 상태 필터 가능)와 실제 동작 불일치
- 개선
- 상태 필터를 항상 활성화하거나, 비활성 사유를 명시
