# 삭제 및 초기화 정책

> 개별 기록 삭제 없이 로컬 초기화와 서버 데이터 전체 삭제를 분리한 정책

## 관련 페이지
- [[book-log]]
- [[feedback]]
- [[timeline-export]]

---

## 목표

- 개별 삭제는 제공하지 않음
- 실수한 기록은 수정과 히스토리로 정정
- 설정 화면에서 `로컬 초기화`와 `서버 데이터 전체 삭제`를 분리

---

## 핵심 원칙

1. **낙장불입**: 사용자용 개별 기록 삭제는 제공하지 않음
2. **수정 우선**: 잘못 남긴 기록은 `PATCH /api/logs/{id}`와 히스토리 누적으로 정정
3. **기기 초기화 분리**: 설정의 `로컬 초기화`는 현재 브라우저 저장소만 삭제하며 서버 데이터는 유지
4. **계정 삭제 분리**: 설정의 `서버 데이터 전체 삭제`는 계정 단위로 서버 기록을 지우고, 현재 기기 로컬 데이터도 함께 초기화

---

## 클라이언트 동작

**로컬 초기화 플로우:**
1. 설정 > 로컬 초기화 → 확인
2. IndexedDB/localStorage 데이터 삭제
3. 새로고침
4. 이후 다시 페어링/동기화하면 서버에 남아 있는 기록은 다시 내려올 수 있음

**서버 데이터 전체 삭제 플로우:**
1. 설정 > 서버 데이터 전체 삭제 → 확인
2. `DELETE /api/auth/account`
3. 서버의 기록/댓글/문의/analytics 이벤트/추천 캐시/기기 연결 삭제
4. 현재 기기 IndexedDB/localStorage 삭제 후 새로고침
5. 이후에는 새 계정 등록 또는 재페어링이 필요

---

## 현재 제품 해석

- `로컬 초기화`는 삭제 기능이 아니라 **현재 기기 비우기**에 가깝다
- `서버 데이터 전체 삭제`는 **계정 단위 전체 삭제**이며 개별 기록 삭제를 대체하지 않는다
- FAQ와 설정 문구는 두 액션의 차이를 명확하게 설명해야 한다

---

## 프론트 체크리스트

- 설정 화면에 "로컬 초기화" 버튼
- 설정 화면에 "서버 데이터 전체 삭제" 버튼
- FAQ에서 "개별 삭제는 없고, 로컬 초기화/서버 데이터 전체 삭제가 분리됨" 안내 문구
- 타임라인/상세/홈에 개별 삭제 진입점 노출 금지
