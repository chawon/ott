# UX 텍스트 일관성 정리 가이드 (2026-02-10)

## 목적
- Web Service + PWA 전반에서 사용자 노출 문구를 동일한 톤/용어/형식으로 운영한다.
- 화면별 임의 문구 추가로 생기는 혼선을 줄이고, 리뷰 기준을 문서화한다.

## 적용 범위
- 대상: `apps/web`의 사용자 노출 텍스트
- 우선 화면: 홈, 타임라인, 함께, 작품 상세, 설정, PWA 설치/오프라인
- 제외: 관리자 전용 페이지(`apps/web/app/admin/**`)의 내부 운영 문구

## 기본 원칙
1. 말투: 기본은 해요체 (`~해요`, `~했어요`, `~할 수 있어요`).
2. 언어: 사용자 노출은 국문 우선, 기술 코드값 영문 노출 금지.
3. 길이: 버튼 2~8자, 안내 문장 12~40자 권장.
4. 상태: 동일 상태는 항상 동일 패턴 사용 (`진행중…`, `완료했어요`, `실패했어요`).
5. 도메인 용어: 서비스 핵심 단어는 고정 사전으로만 사용.

## 용어 사전 (고정)
- `로그`: 저장된 1건 단위 데이터(내부 용어). 사용자 노출은 가급적 `기록` 사용.
- `기록`: 사용자가 입력/저장하는 행위 및 결과.
- `타임라인`: 기록 목록 화면/개념.
- `함께 기록`: 공개 토론/코멘트 영역.
- `작품`: 영상 + 책의 상위 개념.
- 타입 라벨: `영화`, `시리즈`, `책` (영문 `Movie/Series/Book` 금지).

## 케이스별 표준화 규칙

### Case 1) 로딩/진행 상태
- 원칙: 동작 + 진행중 패턴으로 통일
- 표준
- 목록/페이지: `불러오는 중…`
- 저장 액션 버튼: `저장 중…`
- 연결 액션 버튼: `연결 중…`
- 내보내기 액션 버튼: `CSV 만드는 중…`
- 점검 포인트
- `...`와 `…` 혼용 금지, `…`로 통일
- 영어 `Loading...`, `LOADING...` 금지
- 관련 위치
- `apps/web/app/public/[id]/page.tsx:55`
- `apps/web/components/CommentsPanel.tsx:164`

### Case 2) 성공 상태
- 원칙: 결과 + 사용자 가치 + 완료형
- 표준
- 저장 성공: `저장했어요.`
- 연결 성공: `연결했어요.` 또는 `연결 완료했어요.`
- 내보내기 성공: `CSV 파일을 저장했어요.`
- 배너형 성공: `좋아요, n번째 타임라인이 쌓였어요.`
- 금지
- 딱딱한 단문(`완료`, `성공`) 단독 사용

### Case 3) 오류/실패 상태
- 원칙: 사용자 관점 문장 + 필요 시 다음 행동
- 표준
- 불러오기 실패: `불러오지 못했어요. 잠시 후 다시 시도해 주세요.`
- 저장 실패: `저장하지 못했어요. 다시 시도해 주세요.`
- 연결 실패: `연결하지 못했어요. 코드를 확인해 주세요.`
- 금지
- `Error`, `Failed to ...` 원문 노출
- 스택/상태코드 그대로 노출(일반 화면)
- 관련 위치
- `apps/web/app/public/[id]/page.tsx:63`
- `apps/web/app/public/page.tsx:35`
- `apps/web/components/CommentsPanel.tsx:64`

### Case 4) 빈 상태(Empty State)
- 원칙: 현재 상태 + 유도 액션 1개
- 표준
- 기록 없음: `아직 기록이 없어요. 첫 기록을 남겨보세요.`
- 함께 기록 없음: `아직 함께 기록이 없어요.`
- 검색 결과 없음: `검색 결과가 없어요.`
- 금지
- `아직 없어요` 단독 사용(맥락 부족)

### Case 5) CTA 버튼/링크
- 원칙: 동사형, 중복 의미 제거
- 표준
- 저장: `기록 저장`
- 이동: `타임라인 보기`
- 설치: `설치`
- 닫기: `닫기`
- 금지
- 모호한 라벨(`보기`, `실행`, `확인`) 단독 사용

### Case 6) 검색/필터 문구
- 원칙: 입력 대상 + 예시 1~2개
- 표준
- 작품 검색: `작품 검색 (예: 듄, 더 베어)`
- 책 검색: `책 검색 (예: 어린 왕자)`
- 영상 검색: `영상 검색 (예: 듄)`
- 필터 라벨: `콘텐츠`, `상태`, `구분`, `플랫폼`
- 금지
- `비디오`/`영상`/`콘텐츠`를 같은 맥락에서 혼용

### Case 7) 타입/메타 라벨
- 원칙: 도메인 사전 고정 사용
- 표준
- `영화`, `시리즈`, `책`
- `MOVIE/SERIES/BOOK`, `Movie/Series/Book` 금지
- 관련 위치
- `apps/web/components/DiscussionList.tsx:103`
- `apps/web/components/RecoShelf.tsx:29`
- `apps/web/app/public/[id]/page.tsx:103`

### Case 8) 모드별 톤(레트로 vs 일반)
- 원칙: 의미는 동일, 표현만 캐릭터화
- 규칙
- 같은 기능은 같은 핵심 명사 유지 (`기록`, `타임라인`, `설정`)
- 레트로 카피는 장식 표현을 허용하되 상태/오류 문구는 일반 모드와 동등 의미 유지
- 점검 포인트
- 레트로 모드 placeholder와 안내 문장도 동작 의미가 바뀌지 않아야 함

### Case 9) PWA/오프라인 특화 문구
- 원칙: 상태 설명 + 복구 기대치 명확화
- 표준
- 오프라인: `오프라인 상태예요. 네트워크가 복구되면 자동으로 다시 연결돼요.`
- 설치 배너(iOS): `Safari 공유 버튼에서 ‘홈 화면에 추가’를 선택해 주세요.`
- 설치 배너(지원 브라우저): `설치하면 홈 화면에서 바로 열 수 있어요.`

## 구현 운영안
1. 카피 단일 소스 도입
- `apps/web/lib/copy.ts`에 공통 문구 상수화
- 키 예시: `copy.status.loading`, `copy.error.loadFailed`, `copy.type.movie`

2. 화면별 정책
- 페이지: 공통 상태문구만 사용
- 컴포넌트: 버튼/placeholder는 공통 사전에서 가져오기

3. PR 체크리스트
- [ ] 사용자 노출 문구가 해요체인가?
- [ ] 영문 타입/오류 문구가 노출되지 않는가?
- [ ] 동일 상태 문구가 화면마다 같은가?
- [ ] Empty 상태에 다음 행동 문구가 있는가?

4. 리뷰 절차
- 기능 PR마다 `rg -n "Error|Failed|Loading|Movie|Series|Book|\?\?\?" apps/web` 실행
- 결과가 있으면 의도적 사용인지 확인 후 치환

## 단계별 적용 계획
1. 1차(빠른 정리)
- 영어 노출 문구와 `???` 즉시 치환
- 로딩/오류/빈 상태 문구 포맷 통일

2. 2차(구조화)
- `copy.ts` 도입 및 핵심 화면 연결

3. 3차(운영화)
- PR 템플릿에 카피 체크리스트 추가
- 분기별 카피 점검(검색 기반) 수행
