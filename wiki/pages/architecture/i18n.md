## 개요

**완료일:** 2026-03-06

Microsoft Store 등 글로벌 스토어 배포를 위해 한국어(`ko`)와 영어(`en`)를 지원하는 i18n 체계를 구축했다. 향후 추가 언어도 확장 가능한 구조다.

---

## 프론트엔드 구현 (Next.js App Router)

### 라이브러리 및 라우팅
- `next-intl` 사용
- URL 기반 언어 구분: 기본 한국어는 `/...`, 영어는 `/en/...` (`localePrefix: "as-needed"`)
- Next.js `proxy`: 브라우저 언어 감지 → 적절한 locale로 리다이렉트
- 번역 사전: `apps/web/messages/en.json`, `apps/web/messages/ko.json`

### 주요 변경
- `app/` 하위 모든 라우트를 `app/[locale]/`로 이동
- 공통 컴포넌트(Header, Footer, Navigation)와 핵심 화면(홈, 타임라인, 상세, QuickLog) 번역 완료
- `2026-04-19`: 헤더에 `KO` / `EN` 전환 버튼 추가, 현재 보고 있는 경로와 query를 유지한 채 locale만 전환
- `lib/utils.ts` 내 상태/장소/평점 라벨 생성 로직 다국어 지원, 기본값 영문으로 변경

## 백엔드 구현 (Spring Boot)

- 모든 핵심 Controller에서 `Accept-Language` 헤더 수신
- 프론트엔드 `lib/api.ts`에서 현재 locale을 전역 헤더로 전송
- TMDB API 호출 시 언어 파라미터 전달 → 해당 언어 검색 결과/상세 반환
- SyncService: 데이터 동기화 시 타이틀 정보를 해당 언어로 hydration

## 특이사항

- **날짜 포맷**: 언어 설정에 따라 `ko-KR` 또는 `en-US` 형식 동적 적용
- **언어 선택 우선순위**: 첫 진입 시에는 브라우저 언어/locale cookie를 사용하고, 이후에는 헤더 언어 전환 버튼과 `NEXT_LOCALE` cookie가 명시적 선택값 역할을 한다
- **UUID Fallback**: `crypto.randomUUID` 미지원 환경(HTTP 등)을 위해 `Math.random` 기반 폴백을 `lib/utils.ts`에 추가
- **언어 추가 방법**: `messages/<lang>.json` 파일 추가 + `i18n/routing.ts`의 `locales` 배열에 추가

## 관리자 화면은 i18n 라우팅 밖에 둔다

- 관리자 UI는 `/admin`, `/admin/analytics`, `/admin/report`, `/admin/feedback`, `/admin/logout`의 top-level 경로를 사용하고 현재 한국어로 고정한다.
- 기존 `/ko|en/admin/**`는 query를 폐기하고 대응하는 `/admin/**`로 리다이렉트한다. URL token 인증은 지원하지 않는다.
- Cloudflare Access self-hosted application의 기준 path를 `/admin`으로 두어 bare path와 모든 하위 경로를 `Account Member` 정책으로 보호한다.
- edge만 신뢰하지 않고 origin의 Next.js `proxy`가 `Cf-Access-Jwt-Assertion`을 team JWKS로 검증한다. RS256 서명, issuer, audience, `exp`, `nbf`, `type=app`이 모두 유효해야 한다.
- 관리자 server page와 same-origin BFF만 backend `/internal/admin/**`를 `X-Admin-Token`으로 호출한다. 브라우저에는 token을 노출하지 않는다.
- 관리자 root layout은 일반 locale layout과 분리되어 GA4, Clarity, PWA/service worker, 사용자 sync runtime을 로드하지 않는다.
