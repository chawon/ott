# 로컬 데이터 초기화 정책

> 개별 기록 삭제 없이 현재 기기 브라우저 저장소만 초기화하는 정책

## 관련 페이지
- [[book-log]]
- [[feedback]]
- [[timeline-export]]

---

## 목표

- 개별 삭제는 제공하지 않음
- 실수한 기록은 수정과 히스토리로 정정
- 설정 화면의 "로컬 초기화"로만 현재 기기 데이터를 비움

---

## 핵심 원칙

1. **낙장불입**: 사용자용 개별 기록 삭제는 제공하지 않음
2. **수정 우선**: 잘못 남긴 기록은 `PATCH /api/logs/{id}`와 히스토리 누적으로 정정
3. **기기 초기화 분리**: 설정의 "로컬 초기화"는 현재 브라우저 저장소만 삭제하며 서버 데이터는 유지

---

## 클라이언트 동작

**로컬 초기화 플로우:**
1. 설정 > 로컬 초기화 → 확인
2. IndexedDB/localStorage 데이터 삭제
3. 새로고침
4. 이후 다시 페어링/동기화하면 서버에 남아 있는 기록은 다시 내려올 수 있음

---

## 현재 제품 해석

- `로컬 초기화`는 삭제 기능이 아니라 **현재 기기 비우기**에 가깝다
- 계정/서버 데이터 삭제 기능은 사용자 UX로 제공하지 않는다
- FAQ와 설정 문구도 같은 의미를 기준으로 유지한다

---

## 프론트 체크리스트

- 설정 화면에 "로컬 초기화" 버튼
- FAQ에서 "현재 기기만 초기화되고 서버 데이터는 유지" 안내 문구
- 타임라인/상세/홈에 개별 삭제 진입점 노출 금지
