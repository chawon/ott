# 홈 함께 기록 트렌드 보충

## 배경

- 홈의 `요즘 함께 하는 기록들`은 현재 `/api/discussions/latest?limit=6&days=14` 결과를 그대로 표시한다.
- 초기 사용량이 적으면 섹션이 비어 보여 서비스가 활성화되지 않은 것처럼 보일 수 있다.
- 실제 사용자 활동처럼 보이게 만들면 안 되므로, 부족분만 TMDB 트렌드 작품으로 명확히 구분해 보충한다.

## 목표

1. 실제 공개 기록을 최대 6개까지 우선 표시한다.
2. 실제 공개 기록이 6개 미만이면 부족한 개수만 TMDB 주간 트렌드로 채운다.
3. 트렌드 작품은 실제 함께 기록과 다른 배지/문구로 표시한다.
4. TMDB 호출은 백엔드에서만 수행하고, 언어별로 1일 캐시한다.

## 범위

### 포함

- movie/tv 주간 트렌드 혼합 조회
- 한국어/영어 `Accept-Language` 기반 TMDB language 전달
- `providerId` 기준 실제 공개 기록과 트렌드 중복 제거
- 트렌드 카드 클릭 시 QuickLog 검색창에 제목 프리필

### 제외

- 개인화 추천
- 책 트렌드 보충
- 실제 공개 기록처럼 댓글/리액션/작성일을 꾸며 표시하는 UX
- TMDB 트렌드 저장 또는 DB 적재

## 검증 시나리오

1. 실제 공개 기록이 6개 이상이면 TMDB 보충 없이 기존 목록만 표시된다.
2. 실제 공개 기록이 1~5개면 부족한 개수만 트렌드 카드가 표시된다.
3. 실제 공개 기록이 0개면 트렌드 카드 6개가 표시된다.
4. 트렌드 카드를 누르면 QuickLog가 영상 모드로 전환되고 제목 검색어가 채워진다.
