# UX 카피 일관성

> 서비스 전반 사용자 노출 문구의 톤/용어/형식 표준화 가이드

## 관련 페이지
- [[ux-reviews]]

---

## 기본 원칙

1. **말투:** 해요체 (`~해요`, `~했어요`, `~할 수 있어요`)
2. **언어:** 사용자 노출은 국문 우선, 기술 코드값 영문 노출 금지
3. **길이:** 버튼 2~8자, 안내 문장 12~40자 권장
4. **상태:** 동일 상태는 항상 동일 패턴 사용
5. **도메인 용어:** 서비스 핵심 단어는 고정 사전으로만 사용

---

## 용어 사전 (고정)

| 용어 | 설명 |
|---|---|
| `기록` | 사용자가 입력/저장하는 행위 및 결과 (사용자 노출) |
| `로그` | 저장된 1건 단위 데이터 (내부 용어) |
| `타임라인` | 기록 목록 화면/개념 |
| `함께 기록` | 공개 토론/코멘트 영역 |
| `작품` | 영상 + 책의 상위 개념 |
| `영화`, `시리즈`, `책` | 타입 라벨 (`Movie/Series/Book` 금지) |
| `Together` | 영문 메뉴명 (구 "Community" 대체) |

## 케이스별 표준

### 로딩/진행 상태
- 목록/페이지: `불러오는 중…`
- 저장: `저장 중…`
- 내보내기: `CSV 만드는 중…`
- `...` 와 `…` 혼용 금지, `…`로 통일
- 영어 `Loading...`, `LOADING...` 금지

### 성공 상태
- 저장 성공: `저장했어요.`
- 연결 성공: `연결했어요.`
- 내보내기 성공: `CSV 파일을 저장했어요.`
- 배너형: `좋아요, n번째 타임라인이 쌓였어요.`

### 오류/실패 상태
- 불러오기 실패: `불러오지 못했어요. 잠시 후 다시 시도해 주세요.`
- 저장 실패: `저장하지 못했어요. 다시 시도해 주세요.`
- `Error`, `Failed to ...` 원문 노출 금지
- 스택/상태코드 그대로 노출 금지

### 빈 상태 (Empty State)
- 기록 없음: `아직 기록이 없어요. 첫 기록을 남겨보세요.`
- 함께 기록 없음: `아직 함께 기록이 없어요.`
- 검색 결과 없음: `검색 결과가 없어요.`

### CTA 버튼/링크
- 저장: `기록 저장`
- 이동: `타임라인 보기`
- 설치: `설치`
- 닫기: `닫기`
- 모호한 라벨(`보기`, `실행`, `확인`) 단독 사용 금지

### 검색/필터 문구
- 작품 검색: `작품 검색 (예: 듄, 더 베어)`
- 책 검색: `책 검색 (예: 어린 왕자)`

### PWA/오프라인
- 오프라인: `오프라인 상태예요. 네트워크가 복구되면 자동으로 다시 연결돼요.`
- 설치 배너(iOS): `Safari 공유 버튼에서 '홈 화면에 추가'를 선택해 주세요.`

## 구현 운영안

1. `apps/web/lib/copy.ts`에 공통 문구 상수화 (키 예시: `copy.status.loading`, `copy.type.movie`)
2. 페이지/컴포넌트에서 공통 사전에서만 가져오기
3. PR 체크리스트:
   - 사용자 노출 문구가 해요체인가?
   - 영문 타입/오류 문구가 노출되지 않는가?
   - 동일 상태 문구가 화면마다 같은가?
4. 정기 검색 점검: `rg -n "Error|Failed|Loading|Movie|Series|Book|\?\?\?" apps/web`
