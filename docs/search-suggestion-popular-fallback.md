# 검색 추천 인기 작품 보충

## 배경

- 홈의 `요즘 함께 하는 기록들`은 실제 함께 기록이 6개 미만이면 `/api/titles/popular` 기반 인기 작품으로 부족분을 채운다.
- QuickLog 검색창의 빈 검색 추천은 현재 `/api/discussions/latest?limit=6&days=14` 결과만 표시한다.
- 초기 사용량이 적거나 최근 함께 기록이 적으면 검색 추천 패널이 비어 보여 기록 시작 흐름이 약해질 수 있다.

## 목표

1. 검색어가 비어 있고 추천 패널을 여는 경우 실제 함께 기록을 먼저 표시한다.
2. 영상 검색 탭에서 실제 함께 기록이 6개 미만이면 부족분만 인기 작품으로 채운다.
3. 인기 작품은 실제 함께 기록과 다른 섹션/문구로 구분한다.
4. 검색어 입력 후 실제 검색 결과에는 인기 작품을 섞지 않는다.

## 범위

### 포함

- `TitleSearchBox` 빈 검색 추천 패널의 영상 탭 fallback
- 기존 `/api/titles/popular?limit=` 재사용
- 실제 함께 기록과 인기 작품의 provider/providerId 및 제목/연도 기반 중복 제거
- `title_select` analytics의 source 구분

### 제외

- 개인화 추천
- 책 인기 작품 보충
- 새 백엔드 API 추가
- TMDB 인기 작품 DB 저장

## 검증 시나리오

1. 영상 탭에서 실제 함께 기록이 6개 이상이면 인기 작품 섹션이 표시되지 않는다.
2. 영상 탭에서 실제 함께 기록이 1~5개면 부족한 개수만 인기 작품으로 보충된다.
3. 영상 탭에서 실제 함께 기록이 0개면 인기 작품 후보가 표시된다.
4. 책 탭은 기존처럼 실제 함께 기록만 표시한다.
5. 검색어를 입력하면 기존 `/api/titles/search` 결과만 표시된다.
