# Apps in Toss Review Fixes (2026-04-14)

## 범위

- 대상: `apps/web`
- 이슈:
  - 앱인토스 심사 피드백 1: 불필요한 확대·축소 제스처 활성화
  - 앱인토스 심사 피드백 2: 공유 카드 UI 비정상 동작
    - 이미지 저장 미동작
    - 공유하기 버튼이 `준비 중...` 상태로 고정되어 보임

## 영향 범위

- API/DB/Sync 영향 없음
- 프론트 런타임만 수정
- 앱인토스 환경에서만 동작이 달라지고, 일반 웹/PWA는 기존 동작 유지

## 구현 방향

1. 앱인토스 환경 감지 시 초기 `viewport` 메타를 강제로 `maximum-scale=1`, `user-scalable=no`로 교체한다.
2. 공유 카드 저장은 일반 브라우저에서는 기존 `<a download>`를 유지하고, 앱인토스 환경에서는 네이티브 `saveBase64Data` 브리지로 저장한다.
3. 공유 카드 공유는 일반 브라우저에서는 기존 Web Share(files) 우선 정책을 유지한다.
4. 앱인토스 환경에서 Web Share(files)가 불가능하면 텍스트 기반 네이티브 `share` 브리지를 사용한다.
5. 공유 버튼은 미리보기 생성 완료 여부와 무관하게 누를 수 있게 바꾼다. 클릭 시 필요한 리소스를 지연 생성한다.

## 검증 시나리오

1. 앱인토스 도메인/환경에서 페이지 진입 후 핀치 줌이 동작하지 않는다.
2. 공유 카드 바텀시트에서 `이미지 저장` 클릭 시 기기 저장 동작이 실행된다.
3. 공유 카드 바텀시트가 열리자마자 `공유하기`가 `준비 중...`으로 고정되지 않고 클릭 가능하다.
4. 일반 모바일 브라우저에서는 기존 이미지 다운로드와 Web Share(files) 동작이 유지된다.

## 2026-04-14 재테스트 결과

- 핀치 줌 차단: ✅ 확인
- API 통신: ✅ `https://ottline.app/api/...` 기준 정상
- 공유 카드 미리보기: ✅ 앱인토스 내부 캔버스 렌더 fallback으로 정상
- 공유 카드 이미지 저장: ✅ `saveBase64Data` 기준 정상
- 공유하기 버튼: ✅ 더 이상 `준비 중...`으로 고정되지 않음
- 공유 결과: ⚠️ 텍스트 공유만 가능

## 남은 제한사항

- 앱인토스 SDK의 `share` 브리지는 현재 `message: string` 형태의 텍스트 공유만 지원한다.
- 따라서 미니앱 내 `공유하기`는 이미지 파일 첨부가 아니라 텍스트 공유로 동작한다.
- 이미지 공유가 필요하면
  - 앱인토스 SDK에서 파일 첨부형 공유 API가 추가되거나
  - `getTossShareLink(path, ogImageUrl)` 기반 링크 공유 전략으로 별도 전환해야 한다.
