# 삭제 동기화(MVP, 복구 없음) 설계 문서

> 2026-04-13 기준 제품 정책과 불일치하여 폐기. 현재 기준 문서는 `wiki/pages/features/delete-sync.md`.

## 목표
- 개별 삭제는 제공하지 않음
- 설정에서 “로컬 초기화”로만 데이터 제거
- 복구 UX는 제공하지 않음

---

## 핵심 원칙
1) **Soft Delete(tombstone)**: 로컬/서버 모두 `deletedAt` 마킹
2) **Outbox 기반 삭제 동기화**: `op: delete` 전송
3) **LWW + 삭제 우선 보호**: 삭제가 최신이면 되살아나지 않음

---

## 클라이언트 동작
### 로컬 초기화 플로우
1) 설정 > 로컬 초기화 → Confirm
2) IndexedDB/로컬스토리지 데이터 삭제
3) 새로고침

---

## 서버 동작(요약)
