# 공유 카드 (Share Card)

> 기록 저장 직후 SNS 친화적인 이미지 카드를 생성해 공유하는 기능

## 관련 페이지
- [[book-log]]
- [[analytics]]

---

## 목표

- 기록 저장 직후 바텀시트로 공유 유도
- SNS 친화적 이미지 카드 생성 (Story 1080×1920 기준)
- MVP: 템플릿 1(한 줄 감상 카드)만 제공

---

## 주요 결정 사항

| 항목 | 결정 |
|---|---|
| 노출 타이밍 | 기록 저장 후 "공유 카드 만들기" 체크 시만 표시 |
| 평점 표기 | 문구 + 1자리 점수(예: 5.0), 토글로 포함/제외 |
| 메모 | 토글로 포함/제외, 줄바꿈 2줄까지 |
| 워터마크 | `On the Timeline` (레트로 모드 제거 후 단일화) |
| 출력 사이즈 | 1080×1920 (Story)만 |
| 포스터 | 시즌 포스터 우선, 없으면 기본 포스터 |

---

## UX 플로우

1. QuickLog 저장 성공 → 체크된 경우 바텀시트 오픈
2. 바텀시트: 카드 미리보기 + 토글(평점/메모) + 버튼(이미지 저장, 공유하기)
3. 공유 방식
   - 모바일: Web Share API (이미지 파일)
   - 데스크탑: 이미지 다운로드

---

## 템플릿 1 구성

- 상단 70%: 포스터 영역
- 하단 30%: 텍스트 패널(제목/메모/메타 칩)
- 메타 칩: 상태/평점/날짜
- 좌하단: 워터마크

**규칙:**
- 메모: 2줄까지만, 나머지 `…` 처리
- 제목: `:` 뒤에서 줄바꿈
- 날짜: `YYYY.MM.DD` 포맷

---

## 기술 선택

**Satori (vercel/og) 채택** — 서버 사이드 이미지 렌더링
- 장점: 모바일/데스크탑 동일 품질, CORS 이슈 적음
- 단점: CSS 제한, 폰트 파일 필요

서버 이미지 렌더 API: `/api/share-card`
